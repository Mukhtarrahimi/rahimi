Windows PowerShell Commands You Should Know

PowerShell is an advanced scripting language and Command Line Interface (CLI) developed by Microsoft and designed for automating and managing system tasks on Windows-based systems. This tool, first released in 2006, is now an integral part of the Windows management tools suite.

In this article from the Rocket website, we are going to introduce you to 20 practical and useful commands in the PowerShell environment. But before that, let's learn a little more about the key features of PowerShell.

Key features of PowerShell

  • Powerful scripting language: PowerShell based on .NET Framework. Microsoft has built in, which allows it to execute powerful and complex commands using scripts.
  • Access to WMI and COM: PowerShell allows users to access and manage extensive Windows Management Instrumentation (WMI) and Component Object Model (COM), useful for automating administrative tasks in Windows.
  • Object Pipelining: Instead of passing raw text between commands, PowerShell passes data as .NET objects. transmits This allows information to be processed more efficiently during a series of commands.
  • Scripts and modules: Users can create scripts and modules to automate various tasks, allowing them to share and reuse code..
  • Network and Internet Access: PowerShell enables making web requests, working with APIs, and automating network tasks.
  • Extensive management capabilities: From managing users and groups in Active Directory to monitoring and managing the status of system services and processes, PowerShell is a powerful tool for system administrators.

Get-Help [help]

If you are new to PowerShell, you might run into some problems; And in such situations, Get-Help will be your savior. This guide provides information about PowerShell cmdlets, commands, functions, scripts, and workflows. Plus, it's easy to use; You just need to type the Get-Help command along with the command you want details about. For example, you can get information about the Get-Process command by typing Get-Help Get-Process.

Get-Command [gcm]

PowerShell allows you to discover commands and their properties using the Get-Command command. This command displays a list of commands for a specific feature for a specific target, based on your search parameter. You just need to type Get-Command along with your search text in PowerShell. For example, Get-Command *-service* displays commands that contain the word -service. Remember to use asterisks on both sides of your search; Because an important point is not known in search commands.

Invoke-Command [icm]

Invoke-Command is your helper when you want to run a PowerShell command or script (whether locally or remotely, on a single computer or multiple computers).

You must type Invoke-Command along with the command or script and its full path.
For example, invoke a Get-EventLog command using Incoke-Command -ScriptBlock {Get-EventLog system -Newest 50} or, on a remote computer named Server01, using Invoke-Command -ScriptBlock {Get- Run EventLog system -Newest 50} -ComputerName Server01.

Invoke-Expression [iex]

Invoke-Expression executes a command or other expression. If you pass a term or string as its input, this command will first evaluate it, then execute it, but unlike the previous command, it will only execute internally.

To use it, you must enter Invoke-Expression along with your command or expression. For example, you can define a variable called $Command with a string containing the Get-Process command. When you run Invoke-Expression $Command, Get-Process is executed as a command internally on your computer.
