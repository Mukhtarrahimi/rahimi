What is Git

Of course, many good training courses have been produced to learn Git, but in this article I am going to introduce you very quickly to Git and to some extent GitHub.

Definition

Git is a set of commands and facilities that observes the change process of one or more files and gives us the ability to manage different versions of the program we develop. In this way, we can search between different versions of a program and restore them. This process is called Version Control System in a very preliminary and basic way. Git was developed by the creator of Linux, Linus Torvalds.

Git is a program that you can download and use on your local environment. So Git is not a centralized system, it's not like everything is on one server and you send your files to that server, Git can be implemented locally. Of course, you will have the ability to use Git on your personal server as well.

1. Git installation

The first thing we need to do is to install Git on our computer:

If you are using Ubuntu like me, just type sudo apt install git in the terminal. If you are using macOS, just type brew install git in the terminal. If you are in Windows, you should install this program. There are also graphical versions of Git that you can use, but working with the text environment is much more enjoyable.

2. Config Git

When you install the gate for the first time, you need to configure the gate according to your specifications. It means to say who you are so that from now on the events that happen will be saved in your name. To do this, enter the following commands:

git config --global user.name "Arastoo" git config --global user.email test@test.com

Use your name and email instead of Arastoo and test@test.com.
From now on, everything you do in Git will be saved with this name and email.

3. Create a repository

As I mentioned before, Git uses the concept of a repository in order to be able to track the information related to a project. So we need to make a tank. For this, open your terminal and go to your project directory with the cd command. After that enter the following command:

git init

After this, a message similar to the following will appear:

Initialized empty Git repository in /home/me/test/.git/

Now, if you enter a command like ls -a, you will see that there is a secret directory called .git that Git created for you. From now on, all information related to your project, such as commits, etc., will be stored in this directory. We also talk about this case.

Create a simple.txt file in that directory.

4. Checking the status of the gate

Git has a git status command that checks the status of Git. For example, it says what changed, what new things were added, whether everything was updated or not, and things like that. Now let's enter this command once and see what we get back:

git status

Output:

On branch master No commits yet Untracked files: (use "git add ..." to include in what will be committed) simple.txt nothing added to commit but untracked files present (use "gitadd"to track)

This output is telling me that I'm on branchmaster (we'll talk about it).

No committee has been created yet
The important thing about this line that says untracked files: actually this line says that there are a series of files in your test directory that Git doesn't follow, in other words, this simple.txt file is new and Git still doesn't know what to do with it. Now the question that arises is how to tell Git to be interested in this file?

5. GitHub

As I said, Git is not a centralized service. It means that anyone can download a Git and implement it on their local environment or server. One disadvantage of using Git locally is that you cannot share it with others. In order to be able to share Git with other developers, it is very simple to use an online version control system such as GitHub or GitLab. We need remote access for this. So we must have a repository on GitHub and put our own repository on that repository in local mode.

5-1. Registering and creating a repository on GitHub

The first thing you have to do is to register in GitHub and then create a repository. For this, just enter this address and fill the fields related to the repository and finally click Create repository. After that, enter the address of your repository.

5-2. Link to GitHub

Now you need to take the .git address of your repository and connect to the repository in the terminal based on the following code:

git remote add origin https://github.com/arastoonet/roockettest.git

origin is a name for my remote connection with the gate. It can be anything else, but this is what they usually do.

Now that I have made a connection and now it's time to send my git local to github. For this, I use the push command as follows.

git push origin master

I said to send the branch master locally on the remote connection that I made with the name origin. By executing this command, you will be asked for your GitHub authentication credentials, which you must enter correctly.

If you encounter a problem, you can use the -f argument, i.e. force, to push, which can be done as follows:

git push -f origin master

5-3. Clone the project

To get the contents of a repository in GitHub, just use the clone command:

git clone https://github.com/arastoonet/sc.git

Now if I create an update in this Git repository. For example, I added a new file, you don't want to clone the whole project from the beginning. You can use the pull command to bring the new updated files to your local repository:

git pull origin master
